Quick Start

Download the latest release and run the installer. That's it.

Or build from source:

git clone https://github.com/egebulut/ultimate_toolkit_by_egebulut.git
cd ultimate_toolkit_by_egebulut
npm install
npm run tauri:dev

Custom OCR Engine Guide

By default, Ultimate Toolkit uses Tesseract OCR via the leptess Rust crate. You can add custom OCR engines.

Default: Tesseract

English (eng) traineddata is used by default. Install additional languages from the OCR plugin config page inside the app, or manually:

# Tesseract language data is stored at:
%APPDATA%/ultimate-toolkit/tessdata/

# You can download language files from:
https://github.com/tesseract-ocr/tessdata_fast/

Alternative: PaddleOCR

PaddleOCR offers excellent accuracy for Chinese, Japanese, and Korean text. To use it:

  1. Install Python 3.9+ and PaddleOCR: pip install paddleocr
  2. Create a wrapper plugin that calls PaddleOCR via subprocess
  3. Register it in the plugin system

Alternative: TrOCR (Transformers)

For the highest accuracy with printed text, use Microsoft's TrOCR via HuggingFace:

  1. Use the HuggingFace inference API from the AI Chat plugin
  2. Or run locally with pip install transformers torch torchvision
Tip: You can switch between OCR engines in the plugin config. Each engine appears as a separate plugin option.

Custom Plugin Guide

Anyone can build a plugin for Ultimate Toolkit. Here's how:

Plugin Structure

Each plugin lives in src/plugins/<name>/ and consists of:

  1. A PluginDefinition - metadata (id, name, icon, etc.)
  2. A PluginBase subclass - onActivate / onDeactivate
  3. An optional Config component - React component for settings
  4. An optional Floating UI component - overlay window

Example: Simple Hello World Plugin

// src/plugins/hello/index.tsx
import { PluginBase } from "../core/PluginBase";
import type { PluginDefinition } from "../../types";

const definition: PluginDefinition = {
  id: "hello",
  name: "Hello World",
  description: "A simple example plugin",
  icon: "hand",
  version: "0.1.0",
  author: "You",
  hasFloatingUI: false,
};

export class HelloPlugin extends PluginBase {
  definition = definition;
  onActivate() { console.log("Hello!"); }
  onDeactivate() { console.log("Goodbye!"); }
}

export const HelloConfig: React.FC = () => (
  <div>
    <h3>Hello Plugin Config</h3>
    <p>This is your plugin configuration page.</p>
  </div>
);

Register the Plugin

// In src/App.tsx
import { HelloPlugin } from "./plugins/hello";
PluginRegistry.register(new HelloPlugin());

Rust Backend (Optional)

For heavy tasks (file I/O, system calls, ML inference), add Tauri commands in src-tauri/src/plugins/:

// src-tauri/src/plugins/hello.rs
#[tauri::command]
pub fn hello_world(name: String) -> String {
    format!("Hello, {}! From Rust!", name)
}

Then register the command in lib.rs and call it from frontend with invoke("hello_world", { name: "World" }).

AI Provider Setup

Ollama (Local, Free)

1. Download from ollama.com

2. Run: ollama pull llama3.2

3. The app auto-detects Ollama and lists available models

OpenAI

1. Go to platform.openai.com/api-keys

2. Create a new API key

3. Paste it in the AI Chat config

Anthropic (Claude)

1. Go to console.anthropic.com/keys

2. Create an API key (starts with sk-ant-)

3. Paste it in the config

Google Gemini

1. Go to aistudio.google.com/apikey

2. Create a free API key

OpenRouter

1. Go to openrouter.ai/keys

2. One key gives access to 200+ models (GPT-4, Claude, Gemini, etc.)

DeepSeek

1. Go to platform.deepseek.com/api_keys

2. Very affordable pricing

HuggingFace

1. Go to huggingface.co/settings/tokens

2. Create a token with 'read' role

3. Use with models like meta-llama/Llama-3.3-70B-Instruct
